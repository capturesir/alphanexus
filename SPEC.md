# 投資組合管理平台 —— 需求規格書 (SRS)

**文件版本：V3.1（計價模型定案修訂版）**

修訂摘要（V3.0 → V3.1）：本版依架構評審結論，對計價模型作出最終定案並修正若干技術矛盾：

1. 精確定義「原始價格」為拆股已調整、股息未調整之收盤價，公司行動會計改採交易單位數重述（Restatement）模型（取代動態股數敘述，二者數學等價，前者與數據源實際口徑一致）
2. 自動股息於除淨日入帳，支援逐筆修改與全局預扣稅率（預設 0%）
3. 修正 TWR 公式之手續費重複扣減
4. 修正增量更新與拆股事件之衝突規則
5. 期權現金進出定義為外部資金流
6. 明確 LIABILITY 語義
7. FxRateMatrix 以對 USD 持久化加三角換算實現
8. 預計算採前端按位置可加總快取

計價模型直接切換，不保留舊（經調整價）模型；歷史 TWR/MWR 數值將隨之變動，屬預期行為。

## 1. 專案概述與核心目標

本專案旨在建構一個跨市場、多幣種的投資組合管理平台。系統採用手機端優先（Mobile-First）設計，最大兼顧至平板螢幕空間。

本系統的核心架構價值在於：透過原始價格（Raw Price）計價體系、交易單位數重述機制與內部現金流轉模型，徹底解決股息除淨、拆股合股導致的資產淨值扭曲與雙重計算等行業痛點。同時，結合多源備援、增量持久化、後台數據預抓等工程設計，確保系統在處理大量交易與複雜會計指標（TWR/MWR）時，具備極致的效能、韌性與數據精準度。

## 2. 數據層架構與市場數據持久化機制（Resilient Data Layer）

### 2.1 數據源抽象層（DataProvider Interface）

後端須實作通用的行情/匯率介面，每筆金融資產（Security）記錄中必須包含 `dataSource` 欄位，依資產類型進行分流：

- **股票 / 普通 ETF**：預設走 YAHOO Provider（調用 chart 端點）。
- **加密貨幣（BTC/ETH 等）**：主源同走 YAHOO（原生支援 BTC-USD 等代碼）；失敗時降級至 COINGECKO Provider（免 API Key 公開接口）。
- **美股期權**：不對接自動行情源。系統僅支援 MANUAL（手動）模式，由用戶自行輸入交易價與結算價，且一律排除在歷史 K 線與圖表動態追蹤之外（會計處理見 §3.4）。
- **冷門/其他資產**：走 CUSTOM_JSON Provider（§2.5）。

### 2.2 多源自動降級與備援鏈（Fallback Chain）

當主數據源失效（介面變更、頻率限制或封鎖）時，系統須秒級自動降級，確保平台不斷線：

- **行情備援**：YAHOO → 失敗 → 本地歷史庫（過期數據，標記 stale）→ 無庫存時 → STOOQ（免費 CSV 接口）。優先採用過期但口徑一致的本地庫存，避免在「含股息調整 / 不含股息調整」的數據源之間切換造成基準對比口徑跳動。
- **法幣匯率備援**：YAHOO `{CCY}=X` → 失敗 → 本地歷史庫 → Frankfurter / ECB（歐洲央行公開數據；不含 TWD）。

### 2.3 數據源數據契約約束（Data Contract Constraint）【V3.1 修訂】

- 所有實現 DataProvider 介面的行情源，其返回的歷史價格序列中，`close`（收盤價）屬性定義為「拆股已調整（split-adjusted）、股息未調整（dividend-unadjusted）」之收盤價。此定義與 Yahoo chart 端點及 Stooq 之原生口徑一致，全站以此為「原始價格（Raw Price）」之操作型定義。
- 各數據源「適配器（Adapter）」寫入本地 MarketData 表的數值，必須排除股息調整、保留拆股調整；同時須處理特殊報價單位換算（如倫敦 GBp 便士 → GBP、以色列 ILA → ILS）。
- 適配器須同步解析並持久化 `dividends` 與 `splits` 事件序列。
- 系統同時持久化經調整收盤價（Adjusted Close）序列，僅供大盤基準對比（§3.1）使用，嚴禁用於個人組合估值。

### 2.4 數據持久化、增量更新與匯率矩陣【V3.1 修訂】

- **行情增量抓取**：行情數據寫入 MarketData 表。後端查詢本地 `max(date)`，僅向外部 API 請求「最後一根 K 線之後」的缺口數據（含約兩週重疊修復窗）。例外規則：① 偵測到新拆股事件時，因外部源的拆股已調整價格會回溯變動，允許全量重抓或本地按因子重縮放；② 因系統同時持久化經調整價序列供基準對比，新股息事件亦觸發調整價之全量重抓（原始價本身不受新股息影響）。除上述例外，禁止整段重抓歷史序列。
- **歷史匯率持久化（FxRateMatrix）**：後端持久化每日各支援幣種對 USD 之匯率序列；任意交叉匯率（Cross Rate）由 USD 三角換算即時求得，結果與完整交叉矩陣等價而儲存成本由 O(N²) 降至 O(N)。前端切換基準貨幣時的動態歷史折算即基於此。
- **收市後自動預抓**：定時任務於主要市場收市後（預設台北時間凌晨 05:00，可由 `PREFETCH_HOUR` 配置），自動掃描全站用戶現有持倉之資產代碼與自訂源，以禮貌性間隔逐一同步當日最新原始價格、股息與拆股事件，實現用戶翌日「秒開 App」體驗。

### 2.5 通用 JSONPath 自訂數據源

針對冷門資產，系統提供 `/api/custom-sources` 接口，允許配置 JSONPath 爬蟲規則（`date_path` 與 `price_path`），日期格式自動偵測（ISO / Unix 秒 / Unix 毫秒 / MM/DD/YYYY）。

- **前端用戶提示機制**：介面必須顯著提示：「此為自訂第三方數據源，若目標網站結構變更可能導致數據失效。」
- **後端錯誤日誌與監控**：若某自訂源連續 3 次抓取失敗或解析超時，系統須自動記錄錯誤日誌（alerts）並向管理員報警，以便快速維護或通知用戶。
- **安全約束**：自訂 URL 僅允許 http(s) 公網位址，伺服器端須拒絕內網/迴環位址（SSRF 防護）。

## 3. 投資組合會計與資產回報計算架構（業界標準方案）【V3.1 定案】

系統全面採用「原始價格（Raw Price）＋ 內部現金流轉（Internal Transfer）」會計模型，與 Portfolio Performance 等業界標準工具口徑一致。本模型直接取代先前以經調整價估值之實作，不保留舊模型；切換後歷史 TWR/MWR 數值將變動，屬預期行為，須於版本說明中告知用戶。

### 3.1 計價與對比原則

- **個人投資組合**（Portfolio NAV / TWR / MWR / 陰陽燭）：一律採用原始收盤價（§2.3 定義）計算。
- **大盤基準對比**（Benchmark Comparison）：一律採用經調整價（Adjusted Price）計算。因系統無法獲取對比 ETF（如 VT、SPY）的外部配息現金，惟有調整價能代表大盤「總報酬（Total Return）」。此不對稱為刻意設計；用戶端 TWR 為扣費含息淨回報，與基準之毛總報酬對比屬業界慣例，須於說明文件註明。

### 3.2 股息與現金流轉邏輯【V3.1 修訂】

- **自動入帳**：後台同步事件後，系統於除淨日（Ex-Dividend Date）自動產生 DIVIDEND 交易：`稅後金額 = 重述後持股數 × 每股股息 × (1 − 預扣稅率)`。預扣稅率取值順序：該筆交易之用戶逐筆修改值 → 用戶全局「股息預扣稅率」設定 → 預設 0%。自動產生之交易標記為「預估（auto）」，用戶可逐筆修改金額、日期（如改為實際派息日）或刪除；經修改即視為手動交易，後續同步不再覆寫；經刪除者列入跳過清單，不再重生。
- **去重規則**：同一代碼於除淨日 ±14 天內已存在股息交易（手動或自動）者，不再自動產生。
- **資產位置分配**：自動股息按除淨日當時各「資產位置」之持股比例分別入帳至對應位置之現金。
- **會計恆等式**：除淨日股價自然下跌、現金等額增加，總資產（NAV）平滑不變；此為投資組合內部資產形態轉換，不計為外部資金流（No Inflow），故回報率曲線於除淨時刻無虛假升降。
- **與券商對帳**：因預扣稅率與派息時點因人而異，系統現金為趨近值，用戶可透過逐筆修改校正至與券商完全一致（取代 V3.0 之「100% 一致」表述）。

### 3.3 公司行動與拆股/合股會計處理【V3.1 修訂：交易單位數重述】

- **事件監聽**：後台定時任務同步行情時，必須同時解析 `events=split`。
- **交易單位數重述（Transaction Restatement）**：因 §2.3 定義之原始價格已含拆股調整，系統不調整歷史價格，亦不採時間軸上的動態股數，而是於估值時將每筆交易之單位數與單價重述至現今拆股口徑：`重述單位數 = 原始單位數 × F`、`重述單價 = 原始單價 ÷ F`，其中 `F = ∏(交易日之後所有拆合股比例)`。用戶輸入之交易紀錄永遠以「成交當時口徑」保存，不被改寫；重述僅發生於計算層。
- **估值公式**：`Value_t = 重述單位數 × RawPrice_t`。重述單位數 × 重述單價 ≡ 原始金額，故現金扣減、成本基礎與資金流均不受影響；此變更屬內部會計調整，不觸發外部現金流，NAV 平滑、回報率曲線無缺口。
- （說明：本條與 V3.0 之「動態股數」模型數學等價；採重述式係因其與 §2.3 數據契約之拆股已調整口徑直接相容，且估值迴圈內股數恆定、實作更簡。）

### 3.4 期權之會計處理【V3.1 新增】

- 期權不納入歷史曲線與 TWR/MWR 追蹤（§2.1）；其現金進出視為外部資金流：買入期權 = 資金流出（Outflow），賣出/結算 = 資金流入（Inflow）。此使 TWR/MWR 對期權盈虧完全中立，且現金帳與券商實際進出一致。
- 期權於資產列表以最後手動輸入之結算價靜態參與當下總資產結算。
- 因歷史曲線不含期權現值，圖表須顯示註記：「不含期權現值 ±X」，避免用戶誤認圖表末值與資產列表總額不符為錯誤。

## 4. 核心功能模組詳述

### 模組一：用戶認證系統

- **訪客模式（Guest Mode）**：免註冊，數據暫存於本地儲存（Local Storage / IndexedDB / 等效機制）。
- **會員登入（User Login）**：支援雲端同步、修改密碼。代碼自主實作，防範開源授權（AGPL-3.0 / EPL-1.0）污染。

### 模組二：擴展交易與帳戶管理模組（Asset List）

#### 1. 交易活動類型與模型擴展【V3.1 修訂】

每筆買賣交易強制包含 `fee`（手續費/稅費）欄位（預設 0），以計算扣除成本後的「淨回報（Net Return）」。活動類型如下：

- **BUY（買入）/ SELL（賣出）**：買入手續費計入成本基礎；賣出手續費自現金扣除。
- **DIVIDEND（股息）**：計入對應資產位置現金；不計為外部 Inflow（§3.2）。
- **INTEREST（利息）**：現金餘額或債券派息；同上，不計為外部 Inflow。
- **FEE（單獨費用）**：平台費、帳戶管理費等投資損耗；自現金扣除、不計為外部 Outflow（屬投資損耗，自然壓低回報率）。
- **LIABILITY（負債支出）**：融資利息或應付款項；會計處理同 FEE（內部支出，減現金、不入資金流），僅標籤與報表分類不同。

#### 2. 交易明細輸入欄位規範

| 欄位名稱 | 類型 | 說明 / 規範 |
|---|---|---|
| 金融資產代碼 | 字串 | 輸入時觸發下拉選單，支援模糊搜尋（美股/港股/A股/日股） |
| 資產類型 | 選單 | 現金 / 債券 / 股票 / 期權（僅手動輸入結算價）/ 其他 / 股息 / 利息 / 費用 / 負債 |
| 資產位置 | 字串 | 區分券商/銀行（如富途、中銀）。留空系統填「預設」 |
| 交易/結算價格 | 浮點數 | 買賣單價或期權手動輸入的結算價（以成交當時口徑輸入，拆股重述由系統處理） |
| 交易手續費（Fee） | 浮點數 | 用戶輸入之交易成本（手續費、印花稅等），預設 0 |

#### 3. 多幣種現金水平看板

- 列表底部展示各幣種獨立現金餘額，支援因融資或期權保證金產生的負數；另列示「累計股息/利息收入」與「累計費用/負債支出」。
- 提供「基準貨幣」切換，調用本地持久化之對 USD 匯率序列三角換算進行即時與歷史折算。

### 模組三：資產變化圖模組（Performance Chart）

#### 1. 淨回報指標計算與效能優化【V3.1 修訂】

- **時間加權回報率（TWR）**：消除外部資金存取干擾之複合收益率：`TWR = ∏ᵢ ( V₁,ᵢ / (V₀,ᵢ + Cᵢ) ) − 1`。其中 `Cᵢ` 為子期間外部資金流（現金存提與期權現金進出）；手續費、FEE、LIABILITY 均不得計入 C——費用由組合內現金支付，已自然反映於 V，於分子再行扣減將構成重複計算（修正 V3.0 公式）。
- **現金加權回報率（MWR / IRR）**：求解使外部淨現金流折現為 0 之內部收益率（年化）；外部流定義同上。
- **效能設計**（取代 V3.0 後台預計算）：每日資產淨值與外部資金流按資產位置可加總（`V_總 = Σ V_位置`），前端按位置快取每日 `{V, C}` 向量，用戶任意勾選位置組合時相加後即時計算 TWR/MWR，切換延遲 < 500ms。XIRR 迭代於交易筆數超過閾值（建議 5,000 筆）時移入 Web Worker 異步執行，絕不阻塞主線程。後端預計算保留為大規模部署之選配方案。

#### 2. 圖表互動、期權排除與事件標註

- **期權排除規則**：見 §3.4；圖表須顯示「不含期權現值 ±X」註記。
- **手勢操作與形態**：支援移動端單指查價（Tooltip）、雙指縮放（Pinch to Zoom）、左右拖動歷史區間；折線圖、平滑曲線圖、陰陽燭圖（日、週、月、年 K 線，基於原始價）。
- **基準對比（Benchmark）**：輸入代碼（如 VT），拉取對應經調整價，將其回報率曲線與用戶 TWR/MWR 曲線重疊對比。
- **股息與拆股事件標註**：於圖表 X 軸動態標註 Div / Split 圖示，用戶點擊（或十字線停留於事件日）時彈出提示（如「2026-04-15 VT 每股派息 $0.62」「AAPL 1 拆 4」），令原始價、股數與現金變動之對帳完全透明。

### 模組四：資訊平台模組

- **精準推送**：自動提取用戶當前持倉代碼，優先推送直接相關新聞；支援「持倉相關」「熱門財經」「全部資訊」切換。

### 模組五：個人中心模組

- **全局 UI 偏好**：深色/淺色、多語言（繁、簡、英）、漲跌顏色偏好、預設圖表形態。
- **股息設定【V3.1 新增】**：「自動股息入帳」開關（預設開）、「股息預扣稅率（%）」全局設定（預設 0）。
- 自訂數據源管理、CSV 匯入/匯出入口。

## 5. 數據導入與遷移功能（Data Migration）

- **通用 CSV 匯入/匯出**：標準 CSV 範本（含 Fee 與全部活動類型）；匯入須相容 Portfolio Performance / Ghostfolio 風格標頭（`type/symbol/quantity/unitPrice/currency`；DIVIDEND 列之 `unitPrice × quantity` 自動換算為金額），支援「取代」或「合併」兩種模式。

## 6. 非功能性需求與技術指標

- **移動端體驗**：觸控元件點擊熱區 ≥ 44×44 dp；雙指縮放與拖拽圖表渲染幀率穩定 55–60 fps。
- **計算與加載延遲**：切換「資產位置」或「TWR/MWR」之重渲染觸發延遲 < 500ms；首載時間（TTFB）< 300ms。
- **語系分離（i18n）**：全站禁止硬編碼文本；語系內容抽離至 `zh-Hant.json`、`zh-Hans.json`、`en.json` 語系包，由前端啟動時載入；為支援離線/file:// 場景，允許內嵌等效字典作為載入失敗之兜底。

## 7. Tailscale 遠端存取說明

本平台支援透過 Tailscale VPN 從外網安全存取，無需暴露公網端口或設定反向代理。

### 7.1 前置條件

- 伺服器與用戶裝置（手機/平板/電腦）均已安裝 Tailscale 並登入同一帳號
- Tailscale 後台服務運行中（`tailscale status` 可查詢連線狀態）

### 7.2 啟動服務

```bash
# 啟動 WealthLens 伺服器（預設 8080 埠）
node server.js

# 或自訂埠
PORT=3000 node server.js
```

### 7.3 從遠端裝置存取

1. 確認伺服器的 Tailscale IP（例如 `100.97.21.57`）
2. 在遠端裝置瀏覽器輸入：`http://<tailscale-ip>:8080`
3. 即可正常使用所有功能（即時行情、帳戶同步等）

### 7.4 安全說明

- Tailscale 使用 WireGuard 協議加密所有流量，無需額外 HTTPS
- 僅 Tailscale 網路內的裝置可存取，公網無法掃描到該端口
- 建議搭配 `pm2` 進行進程管理，確保服務持續運行：

```bash
npm i -g pm2
pm2 start server.js --name wealthlens
pm2 save
pm2 startup  # 設定開機自啟
```
